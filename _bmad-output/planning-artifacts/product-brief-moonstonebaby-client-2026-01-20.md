# Product Brief: moonstonebaby-client

## Executive Summary

moonstonebaby-client는 쌍둥이를 포함한 다자녀 가정을 위한 통합 육아 플랫폼입니다. 사진 공유와 육아일지(수유, 배변, 수면, 이유식 등) 기록을 하나의 타임라인에서 통합 관리하며, 특히 쌍둥이 관리에 최적화된 UX와 패턴 비교 기능을 제공합니다. 기존 앱들이 겪는 데이터 동기화 버그와 쌍둥이 관리의 복잡성을 해결하여, 부모들이 안정적이고 직관적으로 아이들의 성장을 기록하고 가족과 공유할 수 있도록 합니다.

---

## Core Vision

### Problem Statement

현재 쌍둥이를 키우는 부모들은 육아일지 앱 사용 시 다음과 같은 문제를 겪고 있습니다:

1. **기술적 신뢰성 문제**: 아이를 전환할 때 UI가 제대로 업데이트되지 않는 버그로 인해 데이터 기록의 신뢰성이 떨어집니다.

2. **쌍둥이 관리의 복잡성**: 
   - 두 아이의 기록을 구분해 관리하는 UX가 헷갈리고 직관적이지 않습니다.
   - 동시에 두 아이의 일지를 기록하는 것이 번거롭습니다.
   - 두 아이의 패턴(수유, 수면, 성장 등)을 비교할 수 없어 개별 관리에 어려움이 있습니다.

3. **사진과 일지의 분리**: 사진 공유와 육아일지가 별도 앱이나 기능으로 분리되어 있어, 시간순으로 통합된 추억과 기록을 한눈에 보기 어렵습니다.

### Problem Impact

이러한 문제들이 해결되지 않으면:
- 부모들이 중요한 육아 데이터를 놓치거나 잘못 기록할 위험이 있습니다.
- 쌍둥이의 개별 성장 패턴을 파악하기 어려워 맞춤형 육아가 어렵습니다.
- 사진과 일지가 분리되어 아이의 성장 스토리를 통합적으로 추적하기 어렵습니다.
- 가족 구성원 간 정보 공유가 비효율적이고, 추억 관리가 산발적으로 이루어집니다.

### Why Existing Solutions Fall Short

Research 분석 결과, 현재 시장의 주요 앱들은 다음과 같은 한계가 있습니다:

- **쑥쑥찰칵**: AI 사진 자동 정리와 가족 공유에 특화되어 있으나, 육아일지 데이터 업데이트 버그와 쌍둥이 관리 기능이 부족합니다.

- **베이비타임**: 수유/수면/배변 등 실용적 기록 기능이 강하나, 사진과 일지의 통합 경험이 약하고 쌍둥이 패턴 비교 기능이 없습니다.

- **맘스다이어리**: 사진과 일지를 결합한 형태이지만, 쌍둥이 관리에 최적화되지 않았고 데이터 분석 기능이 제한적입니다.

- **기타 경쟁사들**: 대부분 단일 아이 관리에 최적화되어 있거나, 쌍둥이/다자녀 가정의 특수한 니즈(동시 기록, 패턴 비교)를 충족하지 못합니다.

### Proposed Solution

moonstonebaby-client는 다음 핵심 기능을 제공합니다:

1. **통합 타임라인 뷰**: 사진과 육아일지(수유, 배변, 수면, 이유식 등)를 시간순으로 통합하여 한 화면에서 확인할 수 있습니다.

2. **쌍둥이 관리 최적화**:
   - 두 아이를 명확히 구분하는 직관적인 UI
   - 한 번에 두 아이의 일지를 동시에 기록할 수 있는 기능
   - 두 아이의 패턴(수유 주기, 수면 패턴, 성장 속도 등)을 시각적으로 비교할 수 있는 분석 기능

3. **안정적인 데이터 동기화**: 아이 전환 시 UI가 즉시 업데이트되고, 모든 기록이 실시간으로 안정적으로 저장 및 동기화됩니다.

4. **가족 공유 및 협업**:
   - 엄마, 아빠, 조부모, 보모 등 가족 구성원과 사진 및 일지 데이터를 실시간으로 공유
   - 역할별 접근 권한 설정 (예: 조부모는 사진 보기만, 부모는 모든 기록 가능)
   - 실시간 동기화로 모든 가족 구성원이 최신 정보를 즉시 확인 가능
   - 커스터마이징 가능한 알림 설정 (새 사진 업로드, 일지 기록 등)

### Key Differentiators

1. **쌍둥이/다자녀 최적화**: 시장에서 유일하게 쌍둥이 관리에 특화된 UX와 패턴 비교 기능을 제공합니다.

2. **사진-일지 통합 경험**: 타임라인에서 사진과 일지를 자연스럽게 통합하여, 아이의 성장 스토리를 한눈에 볼 수 있습니다.

3. **기술적 신뢰성**: 데이터 동기화 버그를 해결하여, 부모들이 신뢰하고 사용할 수 있는 안정적인 플랫폼을 제공합니다.

4. **한국 시장 특화**: Research에서 확인된 한국 시장 니즈(예방접종 일정, 한국어 자연어 처리 등)를 향후 반영할 수 있는 확장 가능한 아키텍처를 갖춥니다.

## Target Users

### Primary Users

#### 부모 (데이터 생성자)

**페르소나:**
- **이름**: 예시 - "민수 & 지은 부부"
- **나이**: 20대 후반 ~ 30대 초/중/후반
- **상황**: 쌍둥이 또는 그 이상의 자녀를 동시에 양육하는 부모
- **생활 패턴**: 맞벌이 부부일 수도 있고, 한쪽이 주 양육자일 수도 있음
- **현재 상태**: 쑥쑥찰칵 등 기존 육아 앱을 사용 중이나 데이터 동기화 버그와 쌍둥이 관리의 어려움을 경험

**문제 경험:**
- 기존 앱에서 아이를 전환할 때 UI 업데이트 버그로 인한 데이터 기록의 신뢰성 저하
- 두 아이 이상의 기록을 구분해 관리하는 복잡한 UX
- 두 아이의 패턴을 비교할 수 없어 개별 맞춤형 육아가 어려움
- 사진과 일지가 분리되어 통합적인 성장 스토리 추적이 어려움

**동기와 목표:**
- **데이터 소유권**: 자신의 서버에 직접 데이터를 저장하여 장기적인 데이터 관리와 프라이버시 보장
- **패턴 분석**: 두 아이 이상의 일지 데이터를 비교하여 개별 성장 패턴 파악
- **통합 관리**: 사진과 일지를 하나의 타임라인에서 통합 관리

**성공 기준:**
1. 아이별 사진 업로드 및 타임라인으로 앨범 확인 가능
2. 두 명 이상의 아이들의 일지 데이터를 각각 기록하고, 다른 기기나 계정에서 생성/수정/삭제 동기화 가능
3. 사용자별 적절한 권한 설정 (부모: 모든 권한, 기타 사용자: 주로 읽기, 필요시 추가 권한 부여 가능)

**주요 사용 시나리오:**
- 수유나 기저귀 갈이 같은 이벤트성 기록을 즉시 입력
- 찍어둔 사진을 업로드하고 관리
- 두 아이의 패턴을 비교 분석하여 육아 결정에 활용
- 가족 구성원과 사진 및 일지 데이터 공유

### Secondary Users

#### 보조 사용자 (데이터 소비자)

**페르소나:**
- **유형**: 조부모, 친인척, 친구 등 부모가 초대한 지인 및 가족 구성원
- **관계 설정**: 첫 접속 시 자신의 관계를 설정 (예: 조부, 조모, 친구 이름 등)

**사용 목적:**
- 아이들의 성장 앨범을 공유받아 확인하는 것이 주 목적
- 타임라인별 사진을 확인하는 패턴
- 댓글이나 반응을 통해 소통

**사용 패턴:**
- 부모가 공유한 타임라인 사진을 주기적으로 확인
- 댓글과 반응을 통해 아이들의 성장에 함께 참여
- 읽기 권한 위주로 제한된 접근 (필요시 부모가 추가 권한 부여 가능)

**초대 및 접속:**
- 카카오톡 링크 공유 또는 이메일을 통해 초대받음
- 첫 접속 시 자신의 관계(조부, 조모, 친구 이름 등)를 설정
- 이후 타임라인에서 아이들의 성장 앨범을 확인

### User Journey

#### 부모의 여정

**1. Discovery (발견)**
- 쌍둥이/다자녀 양육의 어려움을 경험하며 기존 앱의 한계를 인식
- 자체 서버에 데이터를 저장하고 패턴 비교가 가능한 솔루션을 찾음

**2. Onboarding (첫 사용)**
- 앱 설치 후 첫 화면에서 아이별 사진이 타임라인으로 표시됨
- 아이 프로필 설정 (쌍둥이 이상의 자녀 등록)
- 권한 설정 (부모는 모든 권한 자동 부여)

**3. Core Usage (핵심 사용)**
- **이벤트 기록**: 수유, 기저귀 갈이 등 즉시 기록
- **사진 관리**: 찍어둔 사진을 업로드하고 아이별로 분류
- **패턴 분석**: 두 아이 이상의 데이터를 비교하여 패턴 파악
- **가족 공유**: 조부모, 친인척 등에게 초대 링크 공유

**4. Success Moment (성공 순간)**
- 두 아이의 패턴 비교 화면을 처음 보았을 때: "이제 각 아이의 개별 패턴을 명확히 알 수 있겠다!"
- 타임라인에서 사진과 일지가 통합되어 보일 때: "아이의 성장 스토리가 한눈에 보인다!"
- 다른 기기에서도 데이터가 완벽하게 동기화될 때: "이제 안심하고 사용할 수 있겠다!"

**5. Long-term (장기 사용)**
- 일상의 필수 도구로 자리잡음
- 아이들의 성장 기록이 축적되며 가족의 소중한 추억 저장소가 됨
- 패턴 분석을 통해 데이터 기반 육아 결정에 활용

#### 보조 사용자의 여정

**1. Discovery (발견)**
- 부모로부터 카카오톡 링크 또는 이메일로 초대받음
- "우리 아이 성장 앨범을 공유해요" 메시지와 함께 초대

**2. Onboarding (첫 접속)**
- 초대 링크를 통해 앱 접속
- 첫 접속 시 자신의 관계 설정 (조부, 조모, 친구 이름 등)
- 읽기 권한으로 자동 설정됨

**3. Core Usage (핵심 사용)**
- 타임라인에서 아이들의 사진을 시간순으로 확인
- 댓글이나 반응을 통해 아이들의 성장에 함께 참여
- 주기적으로 앱을 열어 새로운 사진과 일지 확인

**4. Success Moment (성공 순간)**
- 타임라인에서 아이들의 성장 과정을 한눈에 볼 수 있을 때: "아이들이 이렇게 자라나고 있구나!"
- 댓글로 소통하며 가족의 일원으로 느껴질 때

**5. Long-term (장기 사용)**
- 아이들의 성장 앨범을 정기적으로 확인하는 습관 형성
- 가족 간의 소통 채널로 활용

## Success Metrics

### User Success (부모 / 데이터 생성자)

- **아이별 타임라인 앨범 품질**: 아이별로 사진을 업로드하고, 시간순 타임라인에서 빠르게 찾아볼 수 있다.
- **이벤트 기록의 완결성**: 수유/배변/수면/이유식 등 이벤트를 아이별로 생성/수정/삭제할 수 있고, 기록 누락 없이 관리된다.
- **다자녀(2명 이상) 운영 UX**: 아이 전환/선택이 직관적이며, 기록 구분이 헷갈리지 않는다.
- **패턴 비교 가치**: 두 아이(또는 그 이상)의 패턴 차이를 비교/시각화하여 육아 의사결정에 실제로 도움이 된다.
- **가족 공유 운영**: 부모가 초대/권한 설정을 통해 ‘가족만의 비공개 공간’을 안정적으로 유지한다.

### User Success (보조 사용자 / 데이터 소비자)

- **성장 앨범 소비**: 타임라인에서 사진과(선택적으로) 일지 요약을 쉽게 확인한다.
- **소통 참여**: 댓글/반응을 통해 가족 소통에 참여한다(읽기 기본, 허용 시 참여).

### Product Quality / Reliability (지금 단계에서 중요한 “성공”)

- **전환 시 UI 일관성**: 아이(대상) 전환 시 타임라인/일지 UI가 즉시 올바른 데이터로 갱신된다(잘못된 아이 데이터 노출 0).
- **동기화 신뢰성**: 생성/수정/삭제가 다른 기기/계정에 정확히 반영된다(충돌/유실/중복 최소화).
- **권한/프라이버시 안전성**: 초대 기반 접근 제어가 확실하며, 보조 사용자는 기본적으로 읽기 위주 권한을 가진다(부모가 예외 권한 부여 가능).
- **데이터 소유 및 보관**: ‘우리 가족 데이터는 우리 서버에 저장’ 원칙을 유지한다(백업/내보내기 포함 여부는 추후 결정).

### Business Objectives

- **초기 목표(비공개 가족용)**: 안정적인 기록/동기화와 쌍둥이/다자녀 UX 완성도 확보
- **측정 방식(정성/정량 혼합)**: 가족 내부 사용 만족도, 기록 누락/동기화 문제 발생 빈도, 재방문/사용 지속 여부

### Key Performance Indicators (KPI)

- **현 단계에서는 보류**: MVP가 안정화되고 사용자 흐름/데이터가 쌓인 뒤, 위 Success Metrics를 기반으로 KPI(타겟 수치)를 정의한다.

## MVP Scope

### Core Features

1. **아이별 타임라인 앨범 (사진/동영상)**  
   - 아이별로 미디어를 업로드하고 시간순 타임라인으로 탐색/확인

2. **이벤트 기록 (육아일지)**  
   - 수유/배변/수면/이유식 등 이벤트를 아이별로 **생성/수정/삭제(CRUD)** 가능

3. **동기화**  
   - 여러 기기/계정에서 동일 데이터가 일관되게 반영(생성/수정/삭제 포함)

4. **권한/초대 (비공개 가족용)**  
   - 부모가 초대(카카오톡 링크/이메일) 및 권한(기본 읽기, 필요 시 추가 권한 부여)을 관리

5. **댓글/반응**  
   - 보조 사용자가 타임라인 콘텐츠(주로 사진)에 댓글/반응으로 소통

### Out of Scope for MVP

- **AI 분류/자동화**(예: 자동 태깅, 자동 정리, 요약 등)
- **음성 기록**
- **병원 예약**

### MVP Success Criteria

1. **권한별 공유/동기화 신뢰성**  
   - 일지 데이터가 여러 디바이스/계정에 대해 **권한 범위 내에서** 정확히 공유되고 동기화된다(생성/수정/삭제 포함).

2. **아이 전환 UI 정확성**  
   - 아이(대상) 전환 시 타임라인/일지 화면이 **즉시 올바른 데이터로 업데이트**되어 표시된다(잘못된 아이 데이터 노출 방지).

### Future Vision

- **성장/발달 차트**: 키/몸무게 등 성장 지표를 시각화하고, 기간별 추세를 확인
- **패턴 고도화**: 주간/월간 리포트, 비교 차트 등 “두 아이(이상) 패턴 차이” 분석을 강화
- **사진 자동 정리**: 아이 인식/태깅, 월령/날짜 기반 자동 분류 등 미디어 관리 자동화
- **데이터 내보내기**: 우리 가족 데이터 소유권 강화를 위한 내보내기/백업(형식은 추후 결정)
